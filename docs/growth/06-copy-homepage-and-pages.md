## B. Implementation spec — render the dual axis on the home

The home template (`components/i18n/templates/home-template.tsx`) is SMB-only today. To add the company axis with minimal risk:

1. **Extend `HomeContent`** (`lib/content/types.ts`) with an optional block so other locales don't break:

```ts
businessAxis?: {
  title: string;
  body: string;
  cta: string;            // label
  ctaTarget: PageKey;     // "compare:web-apps-for-business"
  items: { title: string; target: PageKey }[];
};
```

2. **Populate it** in `en.ts` and `pt.ts` `home` objects with the copy above (items → web-apps-for-business, client-portals, internal-dashboards, process-automation, web-apps-for-law-firms).
3. **Render** a new `<Section>` in `home-template.tsx` after the sectors section, only when `content.businessAxis` is set (so ES/FR/DE stay valid until translated). Reuse the existing card styling.
4. **Update the hero** H1/subtitle to the dual-axis lines above (or keep current and add the company axis as the new section — lower risk, ship first, A/B the hero later).

## C. Restaurants page — add sub-page links (fix orphans)

In `industries.restaurants` you can't add links via the current `IndustryContent` shape. Two options:

- **Low-effort:** add a sentence to `solution`/`upgrade` naming the sub-pages (text only).
- **Better:** add an optional `relatedPages?: { label: string; target: PageKey }[]` to `IndustryContent` and render a "Related" list in `IndustryTemplate`; set it on restaurants → `restaurant-reservation-system`, `digital-menu-for-restaurants`, `restaurant-online-orders`. Same pattern recommended for `CompareTemplate` (doc 03 §2.3).
